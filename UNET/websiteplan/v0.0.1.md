# MVP Prompt v0.0.1 - Ground Version

## Goal

Bangun versi paling awal aplikasi web untuk segmentasi flood dengan alur sangat sederhana:

1. Upload 1 gambar.
2. Jalankan model inferensi.
3. Tampilkan hasil mask black-and-white untuk direview.

Tidak ada batch, tidak ada edit brush/eraser, tidak ada smart selection, tidak ada export dataset pada versi ini.

## Product Scope (Strict)

- Input: single image upload (`.jpg`, `.jpeg`, `.png`).
- Process: backend memuat model checkpoint lalu inferensi 1 gambar.
- Output UI:
  - panel kiri: original image.
  - panel kanan: predicted binary mask (hitam-putih).
  - info kecil: waktu inferensi (ms) + threshold yang dipakai.

## Non-Goals

- Multi image upload.
- Queue/background jobs.
- Annotation tools (brush/eraser/polygon).
- Smart selection (SAM/click prompting).
- User authentication.
- Dataset export ZIP.
- Database persistent.

## Suggested Stack

- Backend: FastAPI (Python).
- Frontend: HTML + vanilla JS (minimal) atau React sangat minimal.
- Model runtime: PyTorch.

## Minimal Architecture

- `backend/app.py`: API utama.
- `backend/inference.py`: load model + preprocess + predict + postprocess.
- `backend/weights/`: lokasi checkpoint.
- `frontend/`: 1 halaman upload + preview hasil.

## API Contract (Minimal)

### POST `/predict`

- Request: `multipart/form-data` dengan field `file`.
- Response JSON:
  - `mask_base64`: PNG binary mask (0/255).
  - `threshold`: float.
  - `inference_ms`: number.
  - `width`: int.
  - `height`: int.

## Inference Rules

- Resize image ke ukuran model (mis. 256x256).
- Normalisasi sama seperti pipeline training.
- Output sigmoid lalu threshold default `0.5`.
- Convert ke mask biner:
  - background = `0`
  - flood = `255`

## UI Acceptance Criteria

- User bisa pilih 1 file gambar.
- Setelah klik "Run", tampil loading state.
- Setelah selesai:
  - original image terlihat.
  - mask black-and-white terlihat.
- Jika error (file rusak/model tidak ketemu), tampil pesan error yang jelas.

## Technical Acceptance Criteria

- Endpoint `/predict` berhasil untuk minimal 10 gambar test tanpa crash.
- Semua output mask hanya punya 2 nilai pixel: `0` dan `255`.
- Waktu inferensi tercatat pada response.

## Test Checklist

1. Upload format valid (`jpg/png`) -> sukses.
2. Upload format invalid (`pdf/txt`) -> ditolak.
3. File terlalu besar -> ditolak dengan pesan jelas.
4. Checkpoint hilang -> backend return error yang bisa dipahami.
5. Output mask benar-benar biner (cek unique values).

## Definition of Done

Versi `v0.0.1` dianggap selesai jika alur end-to-end berikut stabil:

- Upload 1 gambar -> inferensi jalan -> mask black-white tampil untuk review.
